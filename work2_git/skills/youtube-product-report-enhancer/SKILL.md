# YouTube 选品报告增强器

在 Python/API 管道已经采集候选品和证据之后，使用本 skill 对 YouTube 选品嗅探报告做大模型增强。

## 工作流程

1. 定位报告数据。
   - 默认项目路径：`reports/latest.json`。
   - 如果用户提供了其它报告文件，以用户提供的文件为准。

2. 构建低 token 模型上下文。
   - 运行本 skill 的 `scripts/build_context.py <report-json> --out <context-md>`。
   - 也可以手动提取同样字段。
   - 上下文只包含 Top 候选品、评分、证据摘要和抽样痛点信号。
   - 不要把原始完整评论流直接交给模型。

3. 增强每个候选品。
   - 合并明显重复或同义的产品名。
   - 把模糊聚类名改写成清晰的产品假设。
   - 用普通电商语言总结用户痛点。
   - 提炼 2-3 个卖点角度和 2-3 个短视频 hook。
   - 补充英国跨境风险：合规、功效宣称、尺码、退货、物流、季节性。
   - 始终把 YouTube 视为内容需求代理，而不是销量证明。

4. 输出最终报告。
   - 使用 `references/enhancement-template.md` 的结构。
   - 按机会质量排序，不要只按 YouTube 分数排序。
   - 保留来源链接，方便用户复核证据。
   - 明确标注弱证据、噪声聚类、或需要人工供应链核查的产品。

## 决策规则

- 优先非食品宠物用品：清洁、出行、消耗精力、家具、替换件、智能护理配件。
- 排除或降权宠物食品、零食、保健品、药品、类药品产品和医疗化宣称。
- 高观看速度但创作者分散度低，视为可能的一次性爆点。
- 多个创作者中等速度覆盖，视为更稳定的趋势。
- 评论里出现具体痛点词，比泛泛夸奖更重要。
- 绝不要仅凭 YouTube 说某个产品已经是爆品或畅销品。

## 输出要求

默认输出简洁但可执行的中文报告。英文产品名可保留，用于供应商搜索、广告检索和竞品研究。

报告必须包含：

- 执行摘要
- Top 产品机会
- 证据解读
- 买家痛点和卖点角度
- 风险和排除说明
- 7 天验证计划
- 下一步需要补充的数据

写最终报告时读取 `references/enhancement-template.md`。解释分数或调整排序时读取 `references/scoring-guide.md`。
